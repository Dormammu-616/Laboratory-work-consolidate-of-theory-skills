Этот проект является часть изучения раздела "ООП" темы "Классы и объекты".
Цель практики: Реализация надежного, защищенного класса Participant (Участник) с применением идиом современного C++ для управления жизненным циклом объектов, инкапсуляции и константной корректности.
🎯 Изученные темы и навыки
В рамках этой работы были отработаны следующие фундаментальные темы ООП:
- Жизненный цикл объекта (RAII): Отслеживание моментов создания и уничтожения объектов через логирование в конструкторе и деструкторе.
- Управление памятью: Работа с объектами на стеке (автоматическая память) и в куче (динамическая память через new/delete).
- Списки инициализации (Member Initializer Lists): Эффективная инициализация полей без лишнего копирования (включая использование тернарных операторов внутри инициализатора).
- Const Correctness: Строгий контроль за неизменяемостью состояния объекта через const методы и аргументы.
- Fluent Interface: Реализация цепочки вызовов (method chaining) через возврат *this.
- Инкапсуляция: Сокрытие данных (private) и предоставление безопасного публичного интерфейса.
🛠 Технические детали реализации
Класс Participant
Класс спроектирован так, чтобы гарантировать целостность данных с момента создания.
- Валидация при создании: Конструктор проверяет входные данные (например, имя) и устанавливает значения по умолчанию прямо в списке инициализации.
- Защита данных: Все поля (id, name, score) приватны. Доступ к ним осуществляется через константные геттеры.
- Методы-цепочки: Сеттеры возвращают ссылку на текущий объект, позволяя писать компактный код:
	```cpp
	obj.set_score(15.5).change_name("New Name");
	```
Демонстрация (main.cpp)
Точка входа демонстрирует разницу в управлении памятью:
- Stack Allocation: Объект создается в локальной области видимости, автоматически уничтожается при выходе из скоупа.
- Heap Allocation: Объект создается динамически. Демонстрируется ручное управление временем жизни и необходимость вызова delete во избежание утечек памяти.
🚀 Как запустить
Проект не требует сторонних библиотек. Для сборки достаточно любого компилятора с поддержкой C++11 и выше.
```bash
g++ main.cpp headers/Participant.cpp -o participant_app -I headers; ./participant_app
```
